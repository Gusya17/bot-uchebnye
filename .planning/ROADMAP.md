# Roadmap: Бот-диспетчер «Анна»

## Overview

Бот строится послойно: сначала инфраструктура (конфиг, БД, состояния FSM) — без неё нельзя начать ни один хендлер; затем полный сценарий текстовой анкеты, который уже даёт владельцу рабочий инструмент приёма заказов; затем отправка брифа (Telegram + email); затем голосовой режим поверх готовой логики; наконец, полировка и подготовка к продакшну. Каждый этап завершает связную, проверяемую возможность.

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: Инфраструктура** — Проект запускается, .env читается, БД создаётся, оба StatesGroup определены, FFmpeg и SMTP проверены
- [ ] **Phase 2: Текстовая анкета** — Полный диалог от /start до подтверждения анкеты клиентом (9 вопросов, валидация, редактирование)
- [ ] **Phase 3: Отправка брифа** — Бриф доставляется владельцу в Telegram и на email, клиент получает подтверждение
- [ ] **Phase 4: Голосовой режим** — Режим «Быстрая заявка»: голос → OGG→MP3 → Whisper → подтверждение расшифровки
- [ ] **Phase 5: Полировка и деплой** — Повторные обращения, глобальные fallback-хендлеры, .env.example, инструкция деплоя

## Phase Details

### Phase 1: Инфраструктура
**Goal**: Каркас бота запущен и настроен: /start отвечает, .env загружается, схема БД создана, оба StatesGroup (TextFormStates, VoiceFormStates) определены, FFmpeg доступен, SMTP-подключение к mail.ru проверено
**Depends on**: Nothing (first phase)
**Requirements**: INFRA-01, INFRA-02
**Success Criteria** (what must be TRUE):
  1. Бот стартует командой `python main.py` без ошибок, отвечает на /start приветствием
  2. Файл .env содержит все секреты; в коде нет ни одного токена, пароля или ключа
  3. База данных SQLite создаётся автоматически при первом запуске, таблица заявок существует
  4. При запуске в лог выводится результат smoke-теста FFmpeg (`ffmpeg -version` успешен) и SMTP-проверки (соединение с smtp.mail.ru:465 установлено)
  5. Оба класса TextFormStates и VoiceFormStates определены в states.py и импортируются без конфликтов

### Phase 2: Текстовая анкета
**Goal**: Студент может пройти полную текстовую анкету от начала до конца: выбрать тип работы, ответить на все вопросы, увидеть итоговую анкету, отредактировать поля, подтвердить или отменить
**Depends on**: Phase 1
**Requirements**: BOT-01, BOT-02, BOT-03, BOT-04, BOT-05, BOT-06, BOT-07, BOT-08, BOT-09, BOT-10, BOT-11
**Success Criteria** (what must be TRUE):
  1. При любом первом сообщении бот отвечает приветствием от «Инги» и предлагает выбрать режим (кнопки: Обычная анкета / Быстрая заявка)
  2. Бот задаёт вопросы строго по одному, показывает прогресс «Вопрос N из 9», принимает файл в поле методических требований
  3. Телефон повторно запрашивается до получения валидного номера; тема из физмат-направления получает вежливый отказ
  4. После 9 вопросов отображается итоговая анкета с кнопками «Подтвердить / Редактировать»; редактирование открывает inline-список полей
  5. Команда /cancel и слова «отмена/стоп» в любой момент сбрасывают FSM и возвращают в начало
**Plans**: TBD
**UI hint**: yes

### Phase 3: Отправка брифа
**Goal**: После подтверждения анкеты клиентом бриф автоматически доставляется владельцу двумя каналами (Telegram DM и email), прикреплённые файлы пересылаются, клиент получает уведомление
**Depends on**: Phase 2
**Requirements**: BRIEF-01, BRIEF-02, BRIEF-03, BRIEF-04, BRIEF-05
**Success Criteria** (what must be TRUE):
  1. Владелец получает личное сообщение в Telegram с отформатированным брифом сразу после подтверждения клиента
  2. На email inga17@mail.ru приходит письмо с тем же брифом (aiosmtplib, port 465, use_tls=True)
  3. Прикреплённые файлы и голосовые из анкеты пересылаются владельцу через file_id без повторной загрузки
  4. Клиент получает сообщение «Заявка принята, свяжемся в течение 2 часов» сразу после отправки брифа
  5. Сбой отправки email не блокирует доставку брифа в Telegram (try/except вокруг SMTP-вызова)
**Plans**: TBD

### Phase 4: Голосовой режим
**Goal**: В режиме «Быстрая заявка» студент может отвечать на каждый вопрос голосовым сообщением: бот расшифровывает его через Whisper, показывает текст, ждёт подтверждения
**Depends on**: Phase 3
**Requirements**: VOICE-01, VOICE-02, VOICE-03, VOICE-04
**Success Criteria** (what must be TRUE):
  1. В режиме «Быстрая заявка» бот принимает голосовое сообщение на каждый вопрос анкеты
  2. Сразу после получения голосового бот отправляет «Расшифровываю...» (пользователь не ждёт в тишине)
  3. Голос конвертируется OGG→MP3 через pydub+FFmpeg и расшифровывается через Whisper API (openai SDK 2.x)
  4. Бот показывает расшифровку и кнопки «Верно / Исправить»; при «Исправить» — запрашивает голос или текст повторно
**Plans**: TBD

### Phase 5: Полировка и деплой
**Goal**: Бот обрабатывает повторные обращения, корректно реагирует на любые неожиданные сообщения, снабжён инструкцией деплоя и .env.example для быстрого старта на сервере
**Depends on**: Phase 4
**Requirements**: INFRA-03
**Success Criteria** (what must be TRUE):
  1. Студент, уже подавший заявку, при повторном /start получает вопрос «Новый заказ или вопрос по текущему?» (inline-кнопки)
  2. Стикеры, геолокации и любые нестандартные типы сообщений получают дружелюбный ответ в тоне «Инги» вместо ошибки
  3. В репозитории есть .env.example со всеми переменными (без значений) и README/инструкция деплоя с шагом `apt install ffmpeg`
