# Requirements — Бот «Инга»

## v1 Requirements

### Диалог и сбор данных

- [ ] **BOT-01**: При любом первом сообщении бот приветствует от имени «Инги» и предлагает выбрать режим (кнопки: Обычная анкета / Быстрая заявка)
- [ ] **BOT-02**: Пользователь может выбрать тип работы из меню (9 пунктов + «Другое»)
- [ ] **BOT-03**: Бот задаёт вопросы строго по одному, ждёт ответа перед следующим
- [ ] **BOT-04**: Бот показывает прогресс (Вопрос N из 9) в текстовом режиме
- [ ] **BOT-05**: Телефон — обязательное поле, бот повторяет запрос до получения
- [ ] **BOT-06**: Клиент может прикрепить файл (PDF, Word, фото) в поле методических требований
- [ ] **BOT-07**: Клиент может написать особые пожелания текстом или голосом в обоих режимах
- [ ] **BOT-08**: После сбора данных бот показывает итоговую анкету и кнопки «Подтвердить / Редактировать»
- [ ] **BOT-09**: Редактирование анкеты через выбор поля (inline-клавиатура), не свободный текст
- [ ] **BOT-10**: Команда `/cancel` и слова «отмена/стоп» сбрасывают FSM и возвращают в начало
- [ ] **BOT-11**: Бот вежливо отказывает при физмат-специализации или теме

### Голосовой режим

- [ ] **VOICE-01**: В режиме «Быстрая заявка» бот принимает голосовые сообщения на каждый вопрос
- [ ] **VOICE-02**: После получения голосового бот сразу отвечает «Расшифровываю...»
- [ ] **VOICE-03**: Голос конвертируется OGG→MP3 и расшифровывается через Whisper API
- [ ] **VOICE-04**: Бот показывает расшифровку и запрашивает подтверждение (Верно / Исправить)

### Отправка брифа

- [ ] **BRIEF-01**: После подтверждения анкеты бот формирует бриф по заданному формату
- [ ] **BRIEF-02**: Бриф отправляется владельцу личным сообщением в Telegram
- [ ] **BRIEF-03**: Бриф отправляется на email inga17@mail.ru
- [ ] **BRIEF-04**: Прикреплённые файлы и голосовые пересылаются владельцу вместе с брифом (через file_id)
- [ ] **BRIEF-05**: Клиент получает подтверждение «Заявка принята, свяжемся в течение 2 часов»

### Инфраструктура

- [ ] **INFRA-01**: Все секреты (токен бота, email, owner_chat_id, openai_key) — только в .env
- [ ] **INFRA-02**: Заявки сохраняются в SQLite
- [ ] **INFRA-03**: При повторном обращении после оформленной заявки — бот уточняет: новый заказ или вопрос

---

## v2 Requirements (отложено)

- Двусторонняя связь: владелец отвечает клиенту через бота
- Статистика заказов / история
- FSM с персистентным хранилищем (Redis/SQLite-backed) для восстановления после перезапуска

## Out of Scope

- Расчёт стоимости — называет только владелец лично
- Физмат/технические работы — явный отказ в боте
- Веб-интерфейс или личный кабинет
- Приём оплаты
- Ответы на вопросы по теме работы прямо в боте
