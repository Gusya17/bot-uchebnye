# PITFALLS — Бот «Анна» на aiogram 3

## 1. FFmpeg не установлен на сервере

**Симптом:** голосовой режим молча падает — бот не отвечает на голосовое сообщение.

**Причина:** pydub требует системный бинарник FFmpeg для конвертации OGG → MP3 перед Whisper. Это не pip-зависимость, она не устанавливается через requirements.txt.

**Профилактика:** Добавить в deployment-инструкцию первым пунктом: `apt install ffmpeg` (Linux) или ручная установка на Windows. Проверять в startup: `subprocess.run(["ffmpeg", "-version"])` и бросать ошибку если не найден.

**Фаза:** Phase 1 (инфраструктура)

---

## 2. Состояние FSM теряется при перезапуске бота

**Симптом:** студент в середине анкеты — бот перезапускается — студент получает приветствие заново как новый пользователь.

**Причина:** `MemoryStorage` хранит FSM-состояния в RAM; при перезапуске всё сбрасывается.

**Профилактика:** Для MVP с малым трафиком это приемлемо. Добавить в `/start` и приветствии проверку: если у пользователя есть незаконченная заявка в SQLite — предложить продолжить или начать заново.

**Фаза:** Phase 1 (инфраструктура)

---

## 3. Устаревший Whisper API в старых туториалах

**Симптом:** `openai.Audio.transcribe()` — AttributeError или 404.

**Причина:** openai SDK обновился до версии 2.x — старый интерфейс сломан.

**Правильно:**
```python
from openai import AsyncOpenAI
client = AsyncOpenAI(api_key=settings.openai_key)
result = await client.audio.transcriptions.create(
    model="whisper-1",
    file=mp3_bytes,
    language="ru"
)
```

**Фаза:** Phase 3 (голосовой режим)

---

## 4. Один StatesGroup для двух режимов

**Симптом:** обработчики текстового и голосового режима начинают срабатывать оба одновременно или в неправильном порядке.

**Причина:** aiogram 3 различает состояния по идентичности класса StatesGroup. Один класс = невозможно зарегистрировать разные хендлеры для одного шага в двух режимах.

**Профилактика:** два отдельных класса `TextFormStates` и `VoiceFormStates` с одинаковыми именами атрибутов.

**Фаза:** Phase 1 (states.py)

---

## 5. Блокировка event loop при отправке email

**Симптом:** бот «замирает» на несколько секунд при отправке брифа.

**Причина:** использование синхронного `smtplib` вместо `aiosmtplib`.

**Профилактика:** только `aiosmtplib`. Никогда не использовать `smtplib` в async-коде.

**Фаза:** Phase 2 (email-сервис)

---

## 6. Хранение файлов вместо file_id

**Симптом:** бот скачивает файл от пользователя, хранит байты в памяти или диске, потом пытается переслать — большой расход RAM, сложность хранения.

**Причина:** непонимание модели Telegram: файлы уже хранятся на серверах Telegram по `file_id`.

**Правильно:** сохранять только строку `file_id` в FSM state и SQLite. Пересылать через `bot.send_document(owner_id, document=file_id)`.

**Фаза:** Phase 2

---

## 7. Отсутствие /cancel — пользователь застрял

**Симптом:** студент хочет начать заново, пишет «стоп», «отмена» — бот игнорирует и снова задаёт текущий вопрос анкеты.

**Профилактика:** глобальный хендлер на команду `/cancel` и на текст «отмена» / «стоп», который очищает FSM-состояние и возвращает в начало. Регистрировать через `Command(commands=["cancel"])` + `StateFilter("*")`.

**Фаза:** Phase 2

---

## 8. mail.ru SMTP: неправильный порт или режим TLS

**Симптом:** `aiosmtplib.SMTPConnectError` или timeout при отправке.

**Причина:** mail.ru требует порт 465 с прямым SSL (`use_tls=True`), а не STARTTLS (587).

**Правильно:**
```python
await aiosmtplib.send(message, hostname="smtp.mail.ru", port=465, use_tls=True,
                      username=settings.email, password=settings.email_password)
```

**Фаза:** Phase 1 (smoke-test инфраструктуры)

---

## 9. Нет индикатора обработки голоса

**Симптом:** пользователь отправил голосовое, ждёт 3–5 секунд тишины и думает, что бот сломался. Отправляет повторно.

**Профилактика:** сразу после получения голосового, до запуска Whisper:
```python
await bot.send_chat_action(chat_id, "typing")
```
И отправить текст «Расшифровываю...» — пользователь видит активность.

**Фаза:** Phase 3

---

## 10. Токен бота в коде

**Симптом:** токен попадает в git-историю → бот компрометирован.

**Профилактика:** только `.env` + `python-dotenv`. `.env` в `.gitignore` с первого коммита. Никогда не коммитить даже «тестовые» токены.

**Фаза:** Phase 1
